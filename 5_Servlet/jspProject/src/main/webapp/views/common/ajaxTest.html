AJAX

서버로부터 데이터를 가져와서 전체 페이지를 새로고침하지않고 일부만 로드할 수 있게 하는 기법 기존에 사용했던 a태그, form submit 방식은 "동기화 요청"이다. => 응답페이지가 돌아와야만 화면을 볼 수 있음

동기식 (a, form submit)

  • 요청처리후 해당 응답페이지가 돌아온 후 다음 작업 가능
  • 서버에 요청한 결과 시간이 지연되면 무작정 계속 기다려야함
  • 전체페이지를 응답해주기 때문에 기본적으로 플리커현상 발생

비동기식 (ajax)

  • 현재 페이지를 그대로 유지하면서 중간중간 필요한 정보를 요청할 수 있는
  • 요청 후 다른 페이지로 넘어가지 않음
  • 요청을 보낸후 해당 응답이 돌아오기 전에 다른작업을 진행할수 있음
  • ex) 검색어 자동완성, 실시간 검색어, 아이디 중복체크, 찜하기/해제, 추천 댓글, 무한스크룰 등등..


비동기식 단점

  • 현재 페이지에 지속적으로 리소스가 쌓임 => 페이지가 느려질 수 있음
  • 오류 발생시 찾기 어려움(디버깅 어려움)
  • 요청 처리후 응답된 데이터를 가지고 현재 페이지에 새로운 요소를 만들어서 추가해줘야 함

AJAX 구현

순수 자바스크립트 방식(XMLHttpReqiest) vs jQuery 방식 (간결하고 사용하기 쉬움) 

        * jQuery 방식

        $.ajax({
            속성: 값,
            속성: 값,
            속성: 값,
            ...
        })

        * 주요 속성
            - url: 요청한 url (필수)
            - type|method: 요청 전송방식 (get/post/delete/put)
            - data: 요청 시 전달할 값
            = success: ajax 요청 성공 시 실행할 함수 정의
            - error: ajax 요청 실패 시 실행할 함수 정의
            - complete: ajax 요청 성공여부와 상관없이 무조건 실행할 함수 정의

            *부가적인 속성
                -async: 서버와의 비동기 처리 방식 설정 여부(기본값 true)
                -contentType: request 데이터 인코딩 방식 (보내는 쪽의 데이터 인코딩)
                -dataType: 서버에서 response로 온 데이터의 형식 설정 (값)
                    xml, json, script, html, text, ...
                - accept: 파라미터의 타입 설장
                - beforeSend: 요청시 실행되는 이벤트 콜백 함수 (데이터 가공, 헤더 설정)
                - cache: 요청 및 결과값을 scope에서 가지고 있지 않도록 하는 설정(기본값 true)
                - contents: jQuery에서 response의 데이터를 파싱한느 방식 정의   
                - context: ajax 메소드 내 모든 영역에서 파싱방식 정의
                - crossDomain: 다른 도메인 호출 가능 여부(기본값 false) 
                - dataFilter: 응답을 받았을 때 정상적인 값을 return할수 있도록 데이터와 데이터타입을 설정
                - timeout: 서버 요청 시 응답대기시간(기본값 0)
