폼(form) 태그

form 태그 내에 submit 버튼 클릭 시 해당 form 태그 내에 작성된
사용자 입력한 값들을 서버로 전달하면서 요청하는 역할을 수행

form action="" method=""

* action : 요청할 서버 경로 => 해당 서버로 사용자가 form 내에 입력 태그를 통해 작성한 값들을 전달
* method : 요청 전송 방식 지정 (get/post). 기본값: get
  > get 방식 : 요청 시 사용자가 입력한 값들이 url에 노출되는 방식
  > post 방식 : 요청 시 사용자가 입력한 값들이 url에 노출되지 않는 방식

form 내의 submit 버튼 클릭 시 action 속성에 지정한 서버로 사용자가 입력한 값들이
전달되면서 요청됨 (사용자가 입력한 값들은 항상 key=value 형태로 전달)

라디오 버튼과 체크박스 관련 input 태그 타입

라디오 버튼

  • type="radio"
  • 제시한 여러 개의 값들 중 한 개만 선택할 수 있도록 하고자 할 때
    • name 속성 값이 같은 요소끼리 하나의 그룹으로 묶여짐
  • form 전송 시(submit) 현재 선택된 라디오 버튼의 value 속성 값이 전달됨
    • 즉, 해당 라디오 버튼 선택 시 전달할 값을 value 속성에 반드시 명시해줘야 함
성별 :

체크 박스

  • type="checkbox"
  • 제시한 여러 개의 값들 중 여러 개 선택할 수 있도록 하고자 할 때
  • 현재 선택된 체크박스의 value값이 전달됨(반드시 value 속성값이 명시되어야 함!)

그 외의 input 태그 타입

type="color"

  • 색상을 선택할 수 있도록 하는 타입

type="file"

  • 파일을 첨부할 수 있도록 하는 타입

type="hidden"

  • 사용자에게 입력받지 않고 화면에 표시도 하지 않으면서 요청 시 전달하고자 하는 값이 있는 경우 사용


버튼 관련 타입들

  • 제출 : submit
  • 초기화 : reset
  • 일반 : button


버튼(button) 태그

  • 시작태그와 종료태그 사이에 화면에 표시할 버튼 내용 작성
  • 용도에 따라 type 속성 사용 (submit/reset/button)

select 태그, option 태그

  • 사용자가 직접 값을 입력하는 것이 아니라 제시된 여러 개의 값들 중 선택하도록 하고자 할 때
  • 제출 시 선택된 option 태그의 값이 전달됨
    • 각 option 태그의 value 속성이 지정되지 않은 경우 content영역의 값이 전달됨
    • 각 option 태그의 value 속성이 지정된 경우 해당 value의 값이 전달됨
